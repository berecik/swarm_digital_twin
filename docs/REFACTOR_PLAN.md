## Future work

### Terrain rendering in the live viewer

The Three.js scene uses a flat grid. Loading SRTM/STL terrain meshes
into the 3D viewport (reusing `terrain.py`'s elevation data) would
give the live view parity with the matplotlib post-flight replayer.

### Real mission thumbnails

Placeholder PNGs generated by Pillow. Replace with screenshots from
real SITL runs when the Docker stack is available.

### Authentication / multi-user

The server binds to `127.0.0.1` only. Exposing on a LAN would need
auth, CSRF protection, and per-user session isolation.

### Swarm SITL live view

`--single-live` works with the SITL stack but swarm SITL modes
(`--swarm`) still fall back to the matplotlib post-flight replayer.
Wiring `sitl_orchestrator.py` to forward telemetry for N drones
via `--telemetry-forward` with per-drone `system_id` would enable
live swarm view from real SITL containers.

### `.BIN` file replay in the web viewer

`POST /api/load` currently supports `.npz` only. Adding ArduPilot
`.BIN` log parsing (reverse of `DataFlashRecorder`) would let
operators replay real SITL flight logs in the browser.
