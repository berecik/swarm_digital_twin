## 1. Terrain Rendering in Live Viewer

**Current state:** The Three.js scene uses a flat grid (`GridHelper`).

**Target:** Load SRTM/STL terrain meshes into the 3D viewport so the live
view matches the simulation terrain.

### Implementation

```javascript
// In live.js — fetch terrain data from a new endpoint
fetch('/api/terrain')
  .then(r => r.json())
  .then(data => {
    // data = { vertices: [...], faces: [...], bounds: [...] }
    const geom = new THREE.BufferGeometry();
    geom.setAttribute('position', new THREE.Float32BufferAttribute(data.vertices, 3));
    geom.setIndex(data.faces);
    geom.computeVertexNormals();
    const mesh = new THREE.Mesh(geom, new THREE.MeshStandardMaterial({
      color: 0x3a5f3a, wireframe: false,
    }));
    scene.add(mesh);
  });
```

**Backend:** Add `GET /api/terrain` to `server.py` that reads from
`simulation/terrain.py`'s elevation grid and returns a simplified mesh.

### Acceptance criteria

- [ ] Terrain mesh renders in `/live` with correct elevation
- [ ] Ground grid is hidden when terrain mesh is present
- [ ] Waypoint poles touch the terrain surface, not y=0

## 4. Swarm SITL Live View

**Current state:** `--swarm` SITL modes use matplotlib post-flight viewer.

**Target:** Forward per-drone SITL telemetry to the live viewer.

### Implementation

In `sitl_orchestrator.py`, when `--telemetry-forward` is set:
- Each `SITLDrone` gets a unique `system_id` (1..N)
- The forward socket tags each frame with the drone's system_id
- `MAVLinkLiveSource` demultiplexes by system_id (already implemented)
- `live.js` renders per-drone meshes (already implemented)

### Remaining work

- Wire `run_swarm_mission_live()` in `run_scenario.sh` (similar to
  `run_single_mission_live()` but for N drones)
- Set `SITL_TELEMETRY_FORWARD` per drone with unique system_id
- Test with 3 and 6 drones

### Acceptance criteria

- [ ] `./run_scenario.sh --swarm-live 3` shows 3 drones in the browser
- [ ] Each drone has its own colour, trail, and label
- [ ] HUD updates from each drone as it becomes the latest sender
